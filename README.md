# OBI LETTERS

読んだ本の感想を手紙として書き、匿名に近い名前で誰か一人に届けるWebサービスのプロトタイプです。

## サービス概要

このアプリは HTML / CSS / JavaScript だけで構成された静的サイトです。
GitHub Pages でそのまま公開でき、URLを知っている人なら誰でもアクセスできます。

## 主な機能

- トップページ（見本の手紙・この場所で大切にしたいこと）
- **ユーザー名記憶機能**（名前を決めて利用開始）
- 手紙を書くフォーム（差出人名・本タイトル・著者名・手紙タイトル・本文）
- 送信前チェックリスト（個人情報・相手を傷つける表現・引用過多などの確認）
- 配達確認注記（初期版では運営が確認してから配達）
- おまかせ便（ランダム受け取り）
- 封筒棚（ランダム順に並ぶ手紙を自分で選ぶ）
- 届いた手紙を読む機能
- 思い浮かんだ一冊を送る機能（返信ではなく、一冊の本の手紙として）
- 手紙箱（受け取った手紙を保管）
- 読後メモ（自分だけの記録）
- 運営への相談（相談内容は運営だけが確認。相手に伝わらない）
- 利用ルール（著作権・精神的健康への配慮を明記）
- お知らせ
- デモ版注記（フッターに表示）
- 管理者用の運営室（手紙一覧・相談一覧・読み手たち一覧・統計含む）
- 運営室でのステータス管理（公開 / 保留 / 非公開 / 削除 / 復元）
- 削除済みの論理削除管理（必要時のみ完全削除）

## 運営室の公開管理

- 手紙は `pending`（未確認）で作成され、運営室で `published`（公開中）にしたものだけが一般画面へ表示されます。
- 公開中の手紙は `status: "published"` の手紙です。運営室の「公開中」タブで一覧確認できます。
- 一般画面（封筒棚 / おまかせ便 / 受け取り）では、`published` 以外の手紙は表示されません。
- `onHold`（保留中）・`private`（非公開）・`deleted`（削除済み）は一般画面から除外されます。
- 削除はまず論理削除（`status: deleted`）として扱い、運営室の削除済みタブで確認できます。
- 削除済みの手紙は復元できます。必要な場合のみ完全削除を実行できます。
- 公開中の手紙を一般表示対象から下げる場合は「非公開にする」を使います。完全に表示対象から外す場合は「削除する」を使います。

## UXの考え方

- 人気順にしない：封筒棚の並びはランダムで、評価されない場を作る
- 一人にだけ届く：たくさんの人へ発信するのではなく、たった一人への手紙
- 安心して使える：個人情報・著作権・精神的安全性に関する説明を随所に配置
- 上手でなくていい：見本の手紙やヒントで、気軽に書けることを伝える

## 使い方

1. このリポジトリを開きます。
2. index.html をブラウザで開きます。
3. 手紙を書く / 受け取る / 手紙箱 / 運営室の動作を確認します。

## GitHub Pagesでの公開方法

1. リポジトリを GitHub に push します。
2. GitHub の対象リポジトリで Settings -> Pages を開きます。
3. Build and deployment の Source を Deploy from a branch に設定します。
4. Branch を main、Folder を / (root) に設定して Save します。
5. 数分待って公開URLにアクセスします。

公開URL例:

https://ユーザー名.github.io/obi-letter/

## 管理者ログインについて

- 管理者入口は画面下部の小さな「運営局」リンクです。
- デモ用パスワードは app.js 内の ADMIN_PASSWORD で管理しています。
- 現在のデモ値は `obi-admin` です。
- 現在の管理機能はプロトタイプ用です。localStorage上のデータを操作しているため、本番環境の管理機能としては不十分です。
- 本番運用では、Firebase、Supabase、MySQL、PostgreSQLなどのデータベースと、安全な管理者認証、権限管理、監査ログが必要です。

## 注意事項

このWebアプリはプロトタイプです。
現在のデータ保存は localStorage を使用しているため、ユーザーごとの端末内にのみ保存されます。
全ユーザーで同じ手紙データを共有する本番サービスとして運用するには、Firebase / Supabase / MySQL / PostgreSQL などのデータベース接続が必要です。

## ユーザー名記憶機能について

このアプリには、プロトタイプ用の簡易ユーザー名記憶機能があります。
ユーザーはメールアドレスやパスワードを登録せず、サービス内で使う名前だけを入力します。
入力されたユーザー名は localStorage に保存され、手紙、読後メモ、相談／通報と紐付けられます。

この方式は、動作確認用の簡易実装です。
本格的なユーザー認証ではありません。
同じブラウザ内でのみユーザー名が保持されます。

全世界のユーザーが同じデータベース上で登録・投稿・受信できるようにするには、
Firebase、Supabase、MySQL、PostgreSQLなどのデータベースと、
安全な認証・ユーザー管理機能を追加する必要があります。

また、管理者パスワードはJavaScript内で管理しているため、本番環境のセキュリティとしては不十分です。
本番公開時には、Firebase Authentication、Supabase Auth、Next.js API Routes などを使い、サーバー側で認証・権限管理を行う必要があります。
